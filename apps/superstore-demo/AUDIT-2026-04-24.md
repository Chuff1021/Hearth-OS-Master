# FireplaceWebstore Audit - 2026-04-24

## Executive Summary

The site builds and typechecks, but it is not yet a real ecommerce/order intake engine. The current storefront looks like a store, but checkout, contact lead capture, newsletter capture, product data source-of-truth, and SEO rendering are not production-ready.

## Verification Run

- `npm install` succeeded.
- `npm run typecheck` passed.
- `npm run build` passed.
- `npm run lint` failed with 3 React hook lint errors.
- `npm audit --audit-level=moderate` found 6 moderate vulnerabilities.
- Vercel production deployment is ready.

## Critical Findings

1. Checkout does not create an order or charge payment.
   - File: `src/app/checkout/page.tsx`
   - The payment form uses raw card-number/CVV inputs and the place-order button is not wired to a payment or order API.
   - This is both a revenue blocker and PCI liability.

2. Contact form is fake.
   - File: `src/app/contact/page.tsx`
   - Submit only sleeps and flips local `submitted=true`; no email, CRM, database, or notification.

3. Newsletter form is fake.
   - File: `src/components/layout/Footer.tsx`
   - No action or submit handler.

4. Public storefront and admin product system are disconnected.
   - Public products come from JSON/sample loaders in `src/app/api/products/route.ts` and `src/lib/*scraped*`.
   - Admin writes to Drizzle tables in `src/db/schema.ts`.
   - Admin product changes may not appear on the storefront.

5. Admin auth is unsafe for production.
   - File: `src/lib/admin-auth.ts`
   - Password hashing is unsalted SHA-256.
   - Default admin may be auto-created from `/api/admin/login` via `ensureDefaultAdmin()`.

6. Missing env vars silently fall back to temporary SQLite.
   - File: `src/lib/env.ts`
   - In production this can create empty/ephemeral state instead of failing safely.

7. Product/category pages are client-rendered and weak for SEO.
   - Files: `src/app/product/[slug]/page.tsx`, `src/app/category/[slug]/page.tsx`
   - Product content loads client-side via `/api/products`; no per-product server metadata or JSON-LD.

8. Placeholder business information.
   - File: `src/lib/store-config.ts`
   - Phone is `(417) 555-0199`, which kills phone conversions and trust.

9. Fake reviews/social proof.
   - File: `src/app/product/[slug]/page.tsx`
   - Hardcoded reviews and review distribution appear on product pages.

10. Runtime uploads write to local/public filesystem.
    - Files: `src/app/api/admin/logo/route.ts`, `src/app/api/admin/upload/route.ts`
    - This will not persist correctly on Vercel serverless; SVG upload also creates XSS risk.

## Revenue / CRO / SEO Findings

1. No real checkout/payment/order system.
2. No lead capture from contact form.
3. No newsletter capture.
4. No analytics events found for product views, add-to-cart, checkout, purchase.
5. No abandoned cart recovery.
6. Hardcoded flat shipping/tax logic in `src/lib/cart-store.ts` is not reliable for heavy fireplace products.
7. Missing sitemap/robots and product structured data.
8. Broken/missing footer pages likely include returns, warranty, installation, showrooms, financing, trade-program, sale, accessibility.
9. Product thumbnail gallery appears interactive but does not switch active image.
10. Quantity can exceed available stock.
11. Parts buyers need fitment confidence: model-number/help CTA is missing.
12. Local SEO pages for Republic/Springfield/Nixa/Ozark/Battlefield are missing.

## Priority Backlog

### P0 - Do Before Real Customers

1. Replace fake checkout with either:
   - Stripe Checkout + order tables + success page, or
   - temporary Request-a-Quote checkout that captures lead/order intent without card fields.
2. Remove raw card inputs until a PCI-safe payment processor is wired.
3. Wire contact form to email/CRM/database.
4. Replace placeholder phone/email/domain details with real business values.
5. Remove fake reviews.
6. Harden admin auth: bcrypt/argon2, no default admin from login endpoint, rate limiting.
7. Fail loud if production DB env is missing.

### P1 - Growth Foundation

1. Server-render PDP/category pages and add `generateMetadata`.
2. Add Product/Offer/Breadcrumb/LocalBusiness/FAQ JSON-LD.
3. Add sitemap and robots.
4. Add GA4/GTM/Meta Pixel ecommerce events.
5. Add returns, warranty, installation, financing, accessibility pages or remove links until ready.
6. Unify product data source: DB-backed storefront or remove misleading admin CRUD.

### P2 - Conversion / Ops

1. Real shipping/freight logic or quote-based freight for heavy items.
2. Abandoned cart capture and email sequence.
3. Fitment/help CTA for parts pages.
4. Product gallery image switching.
5. Quantity clamp and low-stock messaging.
6. Move uploaded assets/product images to durable object storage/CDN.

## Recommended First Implementation Plan

1. Convert checkout from fake card form into a safe quote/order-intent flow immediately.
2. Wire contact form and newsletter capture.
3. Fix store config: real phone, email, address, brand/domain language.
4. Remove fake reviews and broken footer links.
5. Fix lint errors.
6. Then build real Stripe/order system once product/source-of-truth is settled.
